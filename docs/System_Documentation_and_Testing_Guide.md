# EcoTide System Architecture, Workflow & Testing Guide

This document explains the complete, end-to-end working flow of the EcoTide Blue Carbon MRV (Monitoring, Reporting, and Verification) platform, how it is implemented, and how you can manually test every single functionality.

## 1. Technical Implementation Summary
The EcoTide application is built as a highly robust, role-based platform utilizing:
*   **Frontend**: React.js, Tailwind CSS, Vite, Lucide Icons, Recharts (Analytics).
*   **Backend**: Node.js, Express, TypeScript.
*   **Database**: Prisma ORM with SQLite (currently `dev.db`), easily swappable to PostgreSQL.
*   **Security & Auth**: JWT (JSON Web Tokens) for Role-Based Access Control (`ADMIN`, `UPLOADER`, `BUYER`). Bcrypt for password hashing.
*   **Storage**: Cloudinary (for raw image/video storage of Evidence).
*   **Blockchain Integration**: Algorand NodeJS SDK (`algosdk`). 
    * *File: `src/services/algorand.ts` manages the connection to Algorand nodes to mint Carbon Credit ASAs and store Evidence Hashes.*

## 2. End-to-End Workflow & Manual Testing Guide

Follow these exact steps to manually test the full functionality of the application from start to finish. Ensure your backend (`npm run dev` in `/backend`) and frontend (`npm run dev` in `/frontend`) are running.

### 🔑 Test Accounts (Generated by Seed Script)
*   **Admin**: `admin@ecotide.org` (Pass: `password123`)
*   **NGO/UPLOADER**: `ngo@ecotide.org` (Pass: `password123`)
*   **Corporate BUYER**: `buyer@corp.com` (Pass: `password123`)

### Step 1: Project Registration & Evidence Upload (Uploader Role)
**The Implemention:** NGOs log in and register projects. When they upload evidence, the file goes to Cloudinary, and the backend generates a `SHA-256 Hash` of the file to guarantee it isn't tampered with.

**How to test:**
1. Go to `http://localhost:5173` and log out if you are logged in.
2. Log in using `ngo@ecotide.org` / `password123`.
3. Navigate to **New Project** in the sidebar.
4. Fill out the project details (Name, Location, Area, GPS Coordinates map). Submit it.
5. Navigate to **Evidence Upload**.
6. Select the project you just created, attach any random image/PDF, and fill out the GPS tags. Click **Upload**.
7. Wait for the upload success message. Navigate to **Dashboard** to see the new project listed.

### Step 2: MRV Report Submission (Uploader Role)
**The Implemention:** A structured Monitoring, Reporting, and Verification form links the uploaded evidence hash with physical plantation metrics (tree height, count).

**How to test:**
1. While still logged in as the NGO, click **Submit MRV Data** (or navigate to MRV Reports).
2. Fill out the report form, estimating the number of plants and entering a tree height. 
3. *Crucial:* Link the report to the newly created Project. Submit the report.
4. Notice that it will say the report status is **"PENDING"** because an Admin needs to verify it.

### Step 3: Admin Verification & Blockchain (Admin Role)
**The Implemention:** Admins review incoming reports. If approved, the backend reaches out to the Algorand Blockchain. It stores the Evidence Hash on-chain for immutability, and automatically mints **Algorand Standard Assets (ASA)** representing Carbon Credits based on the project's parameters.

**How to test:**
1. Log out of the NGO account.
2. Log in using `admin@ecotide.org` / `password123`.
3. You will land on the **Admin Dashboard**. Notice the analytics charts updating based on the seed data and your new project!
4. Click the **Pending Reviews** tab.
5. You will see the MRV Report you just submitted as the NGO. 
6. Click to expand it or click **Approve** directly.
7. *Behind the scenes:* The backend automatically generates an Algorand Transaction ID, mints tokens (e.g. 5,000 ASAs), and generates an immutable hash proof. The status of the project changes to `VERIFIED`.

### Step 4: Marketplace Purchasing (Buyer Role)
**The Implemention:** Corporate buyers don't use crypto wallets; they click "Buy", simulating a Fiat (Stripe/Razorpay) gateway. Upon successful payment, the backend immediately subtracts the pre-minted Algorand ASAs from the project's supply and assigns them permanently to the Buyer's portfolio.

**How to test:**
1. Log out of the Admin account.
2. Log in using `buyer@corp.com` / `password123`.
3. You will be redirected to the **Marketplace**. 
4. You will see the newly verified project with its total supply of Carbon Credits available.
5. Click **Purchase Carbon Credits**.
6. A prompt will ask you how many you want to buy (e.g., 500). Enter a number and confirm the simulated checkout total.
7. The purchase will succeed. Notice the project's green progress bar instantly shrinks—the available supply has accurately decreased.

### Step 5: Offset Portfolios & PDF Certification (Buyer Role)
**The Implemention:** Buyers need proof for their ESG (Environmental, Social, Governance) sustainability reports. An HTML-to-Print function calculates the exact data, fetches the blockchain TxHash, and spawns a styled PDF Certificate.

**How to test:**
1. While still logged in as the Buyer, click the **My Portfolio & Certificates** tab.
2. You will see your portfolio grid listing the exact transaction you just made.
3. Click the **Download** button next to your transaction.
4. An Official "Certificate of Carbon Offset" will appear in a print dialog window.
5. Notice that it contains your Name, the Project Name, the exact date, the purchased amount, and the heavily secured **Algorand TxHash** proving that the transaction exists on the ledger permanently. Save it as a PDF or close the window.

### Step 6: Audit & User Management (Admin Role)
**The Implemention:** For ultimate compliance, every action taken by an Admin (Approving projects, toggling user access) is strictly permanently written to an `AuditLog` table. 

**How to test:**
1. Log out and return to `admin@ecotide.org`.
2. Go to the **Audit Logs** tab. You will explicitly see the actions "REPORT_APPROVED" and "CREDITS_MINTED" perfectly logged with your admin ID.
3. Go to the **Users** tab. You can instantly Deactivate the Uploader or Buyer accounts with a single toggle switch to block fraudulent actors.
